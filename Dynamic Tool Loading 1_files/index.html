Consuming MCP Servers

Module 3: Dynamic Tool Loading

Static vs. Dynamic Tool Lists

Most agent tutorials hardcode tool definitions in the system prompt. This works for demos but breaks in production.

Static tool lists

  • Tools are defined at compile time in the agent's code or config
  • Add a new tool → redeploy the agent
  • Remove a tool → redeploy the agent
  • Simple, predictable, but brittle

Dynamic tool lists

  • Tools are discovered at runtime from MCP servers
  • Add a new MCP server → agent picks up its tools automatically
  • Remove a server → agent adapts without redeployment
  • More complex, but resilient and composable

Why dynamic matters at scale

In an enterprise environment, the set of available tools changes frequently:

  • New compliance rules require new checking tools
  • Market data providers change or add feeds
  • Internal systems are upgraded or replaced
  • Different teams expose new capabilities as MCP servers

With static tool lists, every change requires a coordinated redeployment. With dynamic discovery, the agent adapts automatically.

Dynamic tool loading is the architectural payoff of MCP. The protocol's discovery mechanism (tools/list) makes it possible to build agents that adapt to their environment at runtime.
