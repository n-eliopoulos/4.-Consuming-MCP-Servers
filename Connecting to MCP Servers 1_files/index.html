Module 2: Connecting to MCP Servers

MCP Client Setup

The ai_shim module provides MCPClient, which mirrors the real MCP SDK's client interface. The simulated client uses the same JSON-RPC 2.0 message format as production MCP, so the code patterns you learn here map directly to real usage.

The initialization handshake

Creating a client and connecting to a server involves a mandatory handshake:

  1. Create an MCPClient pointing at a server name
  2. Call initialize() — the client sends its protocol version and capabilities
  3. The server responds with its own capabilities (which of tools/resources/prompts it supports)
  4. The connection is now ready for use
from ai_shim import MCPClient

client = MCPClient(server="market-data")
info = client.initialize()
print(f"Server: {info['name']}")
print(f"Capabilities: {list(info['capabilities'].keys())}")
Why initialize matters: Not all servers support all capability types. A market-data server might expose tools and resources but no prompts. The initialization handshake tells you what's available before you try to use it.
