Listing Tools

tools/list returns an array of tool definitions. Each tool has three fields that matter:

  • name — The identifier used to invoke the tool (e.g., get_stock_quote)
  • description — What the LLM reads to decide when to use the tool. This is prompt engineering for tools — a vague description means the LLM won't know when to call it.
  • inputSchema — JSON Schema defining the tool's parameters. The LLM uses this to construct valid arguments.
{
  "name": "get_stock_quote",
  "description": "Get the current price, volume, and change for a stock ticker symbol. Returns real-time market data.",
  "inputSchema": {
    "type": "object",
    "properties": {
      "ticker": {"type": "string", "description": "Stock ticker symbol (e.g., AAPL, MSFT)"}
    },
    "required": ["ticker"]
  }
}
Tool descriptions are critical. The LLM decides which tool to call based on the description. A description like "does stuff with stocks" will fail. A description like "Get the current price, volume, and change for a stock ticker symbol" tells the LLM exactly when this tool is useful.
