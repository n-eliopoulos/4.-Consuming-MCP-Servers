Consuming MCP Servers

Module 1: What MCP Solves

What Is MCP?

Model Context Protocol (MCP) defines a uniform way for AI models to discover, invoke, and receive results from external capabilities. It is an open standard created by Anthropic that connects AI applications to external tools and data sources.

Three capability types

Every MCP server exposes capabilities in one or more of these categories:

Mental model: Tools are verbs (do something). Resources are nouns (read something). Prompts are templates (structure something).
  1. Tools — Actions the model can invoke. Examples: get_stock_quote, check_compliance, run_query. Tools take parameters and return results. They are the most common capability type.
  2. Resources — Data the model can read. Identified by URI (e.g., market://indices/current). Resources are read-only data sources: files, database records, API responses. Think of them as "things the model can look at."
  3. Prompts — Reusable prompt templates with arguments. A server can expose pre-built prompt templates that clients fill in with specific values. Example: an analyze-trade-compliance template that takes trade details as arguments.
Note: MCP capability discovery happens at connection time. A server that changes its tool list after initialization without a re-discovery cycle will cause the client to call tools that no longer exist.

Why an open standard matters

Before MCP, every framework had its own tool-calling format. LangChain tools, LlamaIndex tools, and Strands tools were all incompatible. MCP provides a single protocol that any client and any server can implement, regardless of framework or language.

MCP's three capability types — tools, resources, and prompts — are the vocabulary you'll use for the rest of this course. Every MCP interaction maps to one of these three.
