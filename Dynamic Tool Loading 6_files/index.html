Module 3: Dynamic Tool Loading

Exercise: Dynamic Tool Agent

Build an agent that discovers tools dynamically, routes calls to the correct servers, and handles server failures mid-conversation.

Requirements

  1. Discover tools from 3 simulated MCP servers at startup (no hardcoded tool names)
  2. Build a unified tool registry mapping tool names to servers
  3. Pass dynamically-discovered tools to Claude
  4. Handle a multi-turn conversation requiring tools from 2+ servers
  5. Handle one server going offline mid-conversation gracefully

Validation criteria

  • Dynamic discovery — no tool names hardcoded in the agent
  • Correct routing — tool calls go to the right server
  • Graceful degradation — server failure doesn't crash the agent
  • LLM adapts — successfully completes the task using remaining tools
This is the key lab exercise for Module 3. It ties together discovery, routing, and resilience — the three pillars of dynamic tool loading.
