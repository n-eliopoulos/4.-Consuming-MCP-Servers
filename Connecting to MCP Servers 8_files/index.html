Module 2: Connecting to MCP Servers

Exercise: Multi-Server Client

Build an MCP client that connects to 2 simulated servers, discovers their capabilities, invokes tools from each, and handles errors gracefully.

Requirements

  1. Connect to both market-data and compliance-docs servers
  2. List all available tools and resources across both servers
  3. Invoke get_stock_quote from market-data with a valid ticker
  4. Invoke search_regulations from compliance-docs with a search query
  5. Handle a simulated connection failure gracefully (don't crash)
  6. Print formatted results for each operation

Validation criteria

  • Successfully connects to both servers
  • Lists capabilities from both servers
  • Invokes at least one tool from each server
  • Handles simulated errors without crashing
  • Output is well-formatted and readable
Grading: The validator checks that you connect to both servers, list capabilities, invoke tools correctly, and handle errors. All 5 criteria must pass.
