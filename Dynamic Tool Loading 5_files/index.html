Consuming MCP Servers

Module 3: Dynamic Tool Loading

Adapting to Capability Changes

Servers can come and go. Tools can be added or removed. A production agent must handle these changes without crashing.

Periodic re-discovery

Refresh the tool registry on a schedule or when errors occur:

def refresh_registry(servers, registry, clients):
    """Re-discover tools from all servers, handling failures."""
    for name in servers:
        try:
            client = MCPClient(server=name)
            client.initialize()
            clients[name] = client
            for tool in client.list_tools():
                registry[tool["name"]] = {"server": name, ...}
        except ConnectionError:
            # Remove tools from this server
            for tool_name in list(registry):
                if registry[tool_name]["server"] == name:
                    del registry[tool_name]
            clients.pop(name, None)

Handling tool-not-found

If the LLM requests a tool that disappeared (server went offline), return a clear error and let the LLM retry with available tools:

if tool_name not in registry:
    return f"Tool '{tool_name}' is no longer available. Available tools: {list(registry.keys())}"
