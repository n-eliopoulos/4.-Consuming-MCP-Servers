Module 4: Security Boundaries

Exercise: Secure MCP Agent

Build an agent that connects to 3 MCP servers with authentication, audit logging, input validation awareness, and graceful error handling.

Requirements

  1. Authentication: Configure API keys per server. Connect with and without keys to see auth failures.
  2. Audit logging: Log every tool invocation with timestamp, user, server, tool, sanitized arguments, result summary, and latency.
  3. PII redaction: Redact account numbers, names, and IDs from logged arguments.
  4. Input validation: Test with a malicious input (SQL injection in tool arguments) and verify the server rejects it.
  5. Graceful auth failure: Handle authentication errors without crashing the agent.

Validation criteria

  • Auth configured per server (different keys)
  • Audit log captures all tool invocations
  • PII is redacted in logged arguments
  • Malicious input is handled safely
  • Auth failure doesn't crash the agent
This is the capstone exercise. Security is not optional in financial services. Every MCP integration you build at FINRA must include authentication, audit logging, and input validation.
