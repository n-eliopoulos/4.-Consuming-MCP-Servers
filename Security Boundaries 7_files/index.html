Audit Logging for Compliance

Every MCP tool invocation should be logged. At FINRA, regulatory requirements mandate audit trails for all system interactions that touch market data or compliance decisions.

Required log fields

FieldExamplePurpose
timestamp2024-11-15T14:32:07ZWhen the call happened
user_identityexaminer-jsmithWho initiated the action
servercompliance-docsWhich MCP server was called
toolcheck_complianceWhich tool was invoked
arguments (sanitized){"trade_id": "T-***"}What was requested (PII redacted)
result_summary"compliant"Outcome (not full response)
latency_ms142Performance tracking
isErrorfalseWhether the tool call failed

PII redaction

Logs must never contain PII. Redact before logging:

  • Account numbers → ***1234 (last 4 digits)
  • Names → [REDACTED]
  • SSN/TIN → ***-**-1234
  • Full trade details → trade ID only
Open the sandbox to build an AuditLogger wrapper around MCPClient. Every tool call is logged with required fields and PII is redacted from arguments.
