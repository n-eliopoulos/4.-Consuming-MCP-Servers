Invoking Tools

tools/call takes a tool name and arguments. Arguments must conform to the tool's inputSchema. The response contains content — an array of content blocks.

Content block types

  • text — Plain text result (most common)
  • image — Base64-encoded image data
  • resource — Embedded resource reference

Happy path

result = client.call_tool("get_stock_quote", {"ticker": "AAPL"})
# result.content = [{"type": "text", "text": "AAPL: $187.42 (+1.2%) Vol: 52.3M"}]
# result.isError = False

Error path

result = client.call_tool("get_stock_quote", {"ticker": "INVALID"})
# result.content = [{"type": "text", "text": "Unknown ticker symbol: INVALID"}]
# result.isError = True

The isError flag distinguishes tool-side failures from successful results. Both return content. The difference is whether the tool succeeded at its task.

Open the sandbox to invoke tools on the market-data server. Try a valid ticker, then an invalid one. Observe how isError signals tool-side failures without crashing the client.
